#### [<code style={{ fontWeight: 'normal' }}>batch.<b>id</b></code>](#)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>ID!</b></code>](/docs/api/flow/graphql/the-graph/scalars/id.mdx) <Badge class="badge badge--secondary badge--non_null" text="non-null"/> <Badge class="badge badge--secondary " text="scalar"/>

#### [<code style={{ fontWeight: 'normal' }}>batch.<b>block</b></code>](#)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>Block_height</b></code>](/docs/api/flow/graphql/the-graph/inputs/block-height.mdx) <Badge class="badge badge--secondary " text="input"/>

The block at which the query should be executed. Can either be a `{ hash: Bytes }` value containing a block hash, a `{ number: Int }` containing the block number, or a `{ number_gte: Int }` containing the minimum block number. In the case of `number_gte`, the query will be executed on the latest block only if the subgraph has progressed to or past the minimum block number. Defaults to the latest block when omitted.

#### [<code style={{ fontWeight: 'normal' }}>batch.<b>subgraphError</b></code>](#)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>\_SubgraphErrorPolicy\_!</b></code>](/docs/api/flow/graphql/the-graph/enums/subgraph-error-policy.mdx) <Badge class="badge badge--secondary badge--non_null" text="non-null"/> <Badge class="badge badge--secondary " text="enum"/>

Set to `allow` to receive data even if the subgraph has skipped over errors while syncing.
