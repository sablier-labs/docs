#### [<code style={{ fontWeight: 'normal' }}>contract.<b>block</b></code>](#block)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>Block_height</b></code>](/docs/api/streams/graphql/the-graph/inputs/block-height.mdx) <Badge class="badge badge--secondary " text="input"/> \{#block\}

The block at which the query should be executed. Can either be a `{ hash: Bytes }` value containing a block hash, a
`{ number: Int }` containing the block number, or a `{ number_gte: Int }` containing the minimum block number. In the
case of `number_gte`, the query will be executed on the latest block only if the subgraph has progressed to or past the
minimum block number. Defaults to the latest block when omitted.

#### [<code style={{ fontWeight: 'normal' }}>contract.<b>subgraphError</b></code>](#subgraph-error)<Bullet />[<code style={{ fontWeight: 'normal' }}><b>\_SubgraphErrorPolicy\_!</b></code>](/docs/api/streams/graphql/the-graph/enums/subgraph-error-policy.mdx) <Badge class="badge badge--secondary badge--non_null" text="non-null"/> <Badge class="badge badge--secondary " text="enum"/> \{#subgraph-error\}

Set to `allow` to receive data even if the subgraph has skipped over errors while syncing.
